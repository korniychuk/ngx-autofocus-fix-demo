For the simple usage no need to enable .triggerDetectChanges option

Only even controls have enabled autofocus

Without .triggerChangeDetection:

  • You will see the error in the debug console: ExpressionChangedAfterItHasBeenCheckedError
  • Focus losing works not properly

Without .triggerChangeDetection:

  • There is no the error in the console
  • However, focus losing still works not properly

Summary: Independent of ChangeDetectionStrategy you should enable .triggerChangeDetection to work with some components from Angular Material that have wrappers like <mat-form-field>...</mat-form-field>

  • Only even controls have enabled autofocus
  • Works properly without .triggerChangeDetection because have not wrappers like <mat-form-field>...</mat-form-field>
